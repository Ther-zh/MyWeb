# 文章管理指南

## 如何添加文章

本博客网站支持通过添加Markdown文件来上传文章。具体步骤如下：

### 1. 创建Markdown文件

在 `articles` 文件夹下创建一个新的 `.md` 文件，文件名建议使用文章标题的英文或拼音。

### 2. 编写文章内容

文章需要包含前言（Front Matter）和正文两部分。前言部分用于定义文章的元数据，正文部分是文章的实际内容。

#### 前言格式

```markdown
---
title: 文章标题
date: 2026-01-30
summary: 文章摘要
---
```

- `title`：文章标题（必填）
- `date`：发布日期（可选，格式：YYYY-MM-DD，默认为当前日期）
- `summary`：文章摘要（可选，默认为空）

#### 正文格式

正文部分使用标准的Markdown语法，支持以下功能：

- 标题（#、##、###等）
- 列表（有序列表和无序列表）
- 代码块（```语言
代码
```）
- 行内代码（`代码`）
- 引用（> 引用内容）
- 粗体（**粗体**）
- 斜体（*斜体*）
- 链接（[链接文本](链接地址)）
- 图片（![图片描述](图片地址)）

### 3. 运行更新脚本

添加文章后，需要运行 `update_articles.py` 脚本来更新网站的文章数据。

#### Windows系统

```powershell
python update_articles.py
```

#### macOS/Linux系统

```bash
python3 update_articles.py
```

### 4. 查看文章

运行脚本后，打开 `blog.html` 页面即可看到新添加的文章。

## 示例文章

以下是一个完整的示例文章：

```markdown
---
title: 示例文章
date: 2026-01-30
summary: 这是一篇示例文章，用于测试文章更新脚本。
---

# 示例文章

## 什么是示例文章？

示例文章是用于测试和演示的文章，通常包含一些基本的内容结构和格式。

## 示例内容

### 列表示例

- 项目1
- 项目2
- 项目3

### 代码示例

```python
print("Hello, World!")
```

### 引用示例

> 这是一段引用文字，用于展示引用格式。

## 总结

这篇示例文章展示了Markdown的基本语法和格式，包括标题、列表、代码块和引用等。
```

## 注意事项

1. 文章文件必须以 `.md` 结尾
2. 前言部分必须使用 `---` 分隔
3. 运行脚本时需要确保Python已安装
4. 文章内容中的特殊字符会被自动处理，无需手动转义
5. 文章会按照日期降序排列，最新的文章会显示在最前面

## 故障排除

### 脚本运行失败

- 检查Python是否已正确安装
- 检查 `articles` 文件夹是否存在
- 检查Markdown文件格式是否正确

### 文章不显示

- 检查Markdown文件是否在 `articles` 文件夹下
- 检查Markdown文件是否包含前言部分
- 检查 `title` 字段是否填写
- 重新运行更新脚本

### 文章格式错误

- 检查Markdown语法是否正确
- 检查代码块的格式是否正确（需要使用三个反引号包裹）
- 检查引用的格式是否正确（需要使用 `>` 符号）
